What is Garbage Value in Java?

In Java, a garbage value refers to an unpredictable or random value that can appear in a variable when it is not explicitly assigned a value. This can happen when a variable is declared but not initialized or when it is accessed before a value is assigned.

Example:
Imagine you're keeping track of the runs scored by a cricket team, and you have a variable to store the total runs. However, due to an oversight, you forget to assign an initial value to the variable.


Handling Garbage Value

To avoid garbage values, always initialize your variables with meaningful values before using them. In our cricket example, ensure that totalRuns is initialized to zero or the appropriate starting value.


Naming Convention in Java

Java is a case sensitive programming language.
Naming conventions are crucial for creating readable and maintainable code. When everyone follows the same set of rules, it becomes easier for developers to understand each other's code. Consistent and meaningful names also contribute to the overall professionalism and quality of your Java programs.

Advantage of Naming Conventions in Java
By using standard Java naming conventions, you make your code easier to read for yourself and other programmers. Readability of Java program is very important. It indicates that less time is spent to figure out what the code does.

Key Naming Conventions in Java:

1. Classes and Interfaces:

  • It should start with uppercase.
  • It should be noun like- Cars, Phones.
  • Use camelCase (capitalize each word except the first).

    • 2. Methods:

  • It should start with lowercase.
  • It should be verb like- main(), println().
  • Use camelCase.

    • 3. Variables:

  • It should start with lowercase.
  • It should not contain the special characters like- $, _, &.
  • Use meaningful names that describe the variable's purpose.
  • Use camelCase.

    • 4. Constants:

  • It should be in uppercase with underscores for separation.
  • Use descriptive names..

    • 5. Packages:

  • Use lowercase letters.
  • Use a reverse domain name as a prefix to ensure uniqueness.

    • 6. Interfaces:

  • Similar to classes but often named with adjectives.

    • Real-World Analogy:
    Consider name conventions to be a language's grammatical rules. Following naming conventions guarantees your code is understandable, just as using proper syntax makes phrases understandable. Imagine trying to follow a tale where characters have abruptly changed their names; it would be quite confusing. Similar to this, code has smooth and pleasant reading due to uniform naming conventions.

    Type Casting in Java

    Type casting is when you assign a value one primitive data type to another primitive data type.
    In Java, type casting refers to the process of converting one data type into another. This is necessary when you want to assign a value of one data type to a variable of another data type, and they are not automatically compatible. Java supports two types of type casting: implicit (automatic) casting and explicit (manual) casting.


    Implicit Type Casting:

    Implicit type casting, also known as automatic casting, happens when Java automatically converts a lower data type to a higher data type without the need for explicit instructions. It is considered safe as it does not result in loss of data. Implicit type casting is also called Up Casting.
    JVM(Java Virtual Machine) will handle this type casting automatically.



    Explicit Type Casting:

    Explicit type casting, also known as manual casting, requires the programmer to explicitly specify the conversion from a higher data type to a lower data type. It may result in loss of data.
    Implicit type casting is also called Down Casting.
    JVM(Java Virtual Machine) will not handle this type casting.


    Real-World Analogy:
    A person moving from a one-bedroom apartment to a two-bedroom apartment will naturally have more space and be able to fit all of his belongings with ease. We refer to this as Implicit type casting. However, if we move someone else from a two-bedroom apartment to a one-bedroom apartment, they won't be able to accommodate all they own in the tiny area. We refer to this as Explict Type Casting.

    User Input in Java

    When a Java application can take input from the user while it is running, it is said to have user input. The software becomes more dynamic and responsive as a result of human interaction and input. User input may be read from the console in Java using the Scanner class.
    In Java, there are 2 types of method for input.

  • Command line argument.
  • Using Scanner Class.

    • Command line argument:
    The parameter given to the Java program during execution is the Java command-line argument.
    The Java program can receive the arguments given by the console and use them as input.

    Using Scanner Class:
    The Java Scanner class enables console input from the user. It is a part of the package java.util. Primitive types such as int, double, long, short, float, and byte are read by it. It is the simplest way for a Java program to read input. 

                             
                                import java.util.Scanner;

    Example:

  • We import the Scanner class from the java.util. package.
  • We create a Scanner object named sc to read input from the console (System.in).
  • We prompt the user to enter their name using System.out.print.
  • We use sc.nextLine() to read the entire line of text entered by the user..
  • We display a greeting message using the user's input.

    • Operators in Java

    Operators are unique symbols or keywords that operate on variables and values in Java. They let you work with data, run calculations, and make choices for your programs.

    Java supports a wide range of operators, which can be categorized into several types:
    1. Arthmetic Operators:

  • + (addition operators).
  • - (substraction operators).
  • * (multiplication operators).
  • / (division operators).
  • % (modulus operators).

    • 2. Comparison (Relational) Operators:

  • == (equal to operators).
  • != (not equal to operators).
  • > (greater than operators).
  • < (less than operators).
  • >= (greater than or equal to operators).
  • <= (less than or equal to operators).

    • 3. Logical Operators:

  • && (logical AND operators).
  • || (logical OR operators).
  • ! (logical NOT operators).

    • 4. Assignment Operators:

  • = (assignment operators).
  • += (addition assignment operators).
  • -= (subtraction assignment operators).
  • *= (multiplication assignment operators).
  • /= (division assignment operators).
  • %= (modulus assignment operators).

    • 5. Increment and Decrement Operators:

  • ++ (increment operators).
  • -- (decrement operators).

    • 6. Bitwise Operators::

  • & (bitwise AND operators).
  • | (bitwise OR assignment operators).
  • ^ (bitwise XOR operators).
  • - (bitwise NOT operators).
  • << (left shift operators).
  • >> (right shift operators).
  • >>> (unsigned right shift operators).

    • Short-hand in Java:
    Short-hand operators is a combination of assignment and arithmetic opeators.
